API Supported Methods


Method Route Functionality
[GET] / Any Get Request made to the root path of the server will return a HELP Page written in HTML and CSS which will outline the functionality of the API. It will list each method that is supported and provide an example of the input and output expected as well as any error paths or error handling that is available.
[POST] /Users When a POST request is made to /Users it will expect a user object as shown below:
{
 "userId": "string",
 "firstName": "string",
 "lastName": "string",
 "emailAddress": "string",
 "password": "string"
}

It should take this data and create a new User record and store it in an in-memory array. All the fields are required and there should be error checking in place for this. If fields are missing this should return a 406 Unacceptable status along with a message. No two users can share the same user ID. If the same user ID is attempted to be created it should respond with 409 Conflict status and a message.
{
 "message": "string",
 "status": "string"
}

If everything is successful it should respond with status 201 (Created) and a copy of the object (without the password)
[GET] /Users Should return a list (JSON ARRAY) of all the Users which have been created with a 200 response or just a blank array.
{
 "userId": "string",
 "firstName": "string",
 "lastName": "string",
 "emailAddress": "string",
}
[GET] /Users/{userId} If the user ID provided exists it should return that user object with a 200 response
{
 "userId": "string",
 "firstName": "string",
 "lastName": "string",
 "emailAddress": "string",
}

If the user ID provided is not found it should return a 404 (not Found) error along with a message. {
 "message": "string",
 "status": "string"
}
[PATCH] /Users/{userId} This method should expect a payload of a user record (incomplete or complete) and should update the existing user with the provided values if the user exists.
{
 "userId": "string",
 "firstName": "string",
 "lastName": "string",
 "emailAddress": "string",
 "password": "string"
}

If the user doesn't exist it should return 404 not found along with a message. {
 "message": "string",
 "status": "string"
}

It should ignore any changes to the user ID but all other fields provided should be updated and a status 200 should be returned along with the updated user object.
[DELETE] /Users/{userId} This request should take the provided user id and remove it from the user array (if found) and respond with status 204 Deleted. If the user is not found it should respond with a 404 not found and appropriate message.
{
 "message": "string",
 "status": "string"
}
